# Demo readiness

## Required before sharing a preview

- Keep the preview private or password protected.
- Confirm every screen shows the concept/sample-data disclosure.
- Run `npm test` and manually complete both game journeys on desktop and mobile.
- Reset browser prototype data before a new stakeholder session when a clean entry is needed.
- Tell viewers that entries live only in their browser and deadlines use their device clock.

## Known prototype limits

- No authentication, server-side database, admin tools or live racing feed.
- Runner fields, schedule, prizes, leaderboards and entry numbers are illustrative.
- Scratchings and race status are modelled in the UI but are not connected to an official source.
- AFL, NRL, NFL and community tiles are non-interactive presentation context.

## Production gates

- Approved rules, eligibility, prize terms and responsible-gaming/legal review.
- Server-authoritative account, entry, deadline and audit-trail services.
- Official race fields, scratchings and result ingestion with operational fallbacks.
- Privacy, security, accessibility and load testing.
- Written approval for every logo, font and supplied image used in the release.
